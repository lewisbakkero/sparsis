## Executive Summary
BoxPromptIML introduces a weakly-supervised image manipulation localization framework that reduces annotation costs from 23 minutes per image to just 7 seconds while achieving competitive localization accuracy. By using bounding boxes as prompts for Segment Anything Model (SAM) to generate pseudo-masks and employing a memory-guided fusion module, it enables efficient training of a lightweight student model without requiring pixel-level annotations.

## Why This Matters for Practitioners
If you're maintaining image forensics systems in production, BoxPromptIML solves the critical annotation bottleneck that's been holding back real-world deployment. The 98% reduction in annotation time (from 23 minutes to 7 seconds per image) means you could process 19x more images with the same annotation team, while the lightweight model (5.5M parameters, 1.4G FLOPs) fits within the constraints of edge devices or mobile applications. For example, if you're currently using a fully-supervised model like Mesorch (85.8M parameters), you could reduce inference latency by 90% and deployment costs by 95% while maintaining comparable accuracy (0.619 F1 score vs. 0.703 for Mesorch on IND datasets). This makes IML practical for real-time applications like social media content moderation, where processing speed and cost efficiency are as important as accuracy.

## Problem Statement
Imagine trying to identify a single misspelled word in a 100-page novel by manually marking every single character, this is the current reality of image manipulation localization. You're spending 23 minutes per image meticulously tracing every pixel of tampered regions, which is as inefficient as trying to find typos by hand in a book. Now imagine having a magnifying glass that lets you circle the general area of the error (taking just 7 seconds), and then a smart tool that uses that circle to automatically highlight all the misspellings. That's the problem BoxPromptIML solves: it eliminates the need for tedious pixel-level annotations while preserving spatial localization cues.

## Proposed Approach
BoxPromptIML uses a two-stage framework where a frozen SAM (Segment Anything Model) serves as a teacher model to transform coarse bounding box annotations into high-quality pseudo-masks, which are then used to supervise a lightweight student model through knowledge distillation. The student model, trained without pixel-level ground truth, learns to localize manipulation directly from input images. The key innovation is the Memory-Guided Gated Fusion Module (MGFM), which dynamically combines real-time feature analysis with recalled prototypical manipulation patterns from a memory bank.

```python
def box_prompt_iml(image):
    # Teacher model (frozen SAM) generates pseudo-mask from bounding box
    pseudo_mask = sam.generate_mask(image, bounding_box)
    
    # Student model predicts manipulation mask from input image only
    student_prediction = student_model(image)
    
    # Memory-Guided Gated Fusion Module refines prediction
    refined_prediction = mgfm_refine(student_prediction)
    
    return refined_prediction
```

## Key Technical Contributions
The authors' main technical contributions are:

1. **Coarse-to-Fine Annotation Strategy**: Instead of requiring pixel-level masks, they use bounding boxes as prompts for SAM, which generates pseudo-masks. This reduces annotation time from 23 minutes to 7 seconds per image while preserving spatial information. The SAM model's zero-shot segmentation capability transforms these low-cost prompts into high-quality supervision signals without manual intervention.

2. **Memory-Guided Gated Fusion Module (MGFM)**: The MGFM maintains a learnable memory bank storing prototypical manipulation patterns (e.g., common splicing artifacts from training data), and employs a gating mechanism to selectively fuse multi-scale features based on their relevance. This dual-guidance strategy (real-time cues + long-term memory) enables dynamic knowledge recollection, enhancing accuracy and robustness without requiring additional annotations.

3. **Efficient Knowledge Distillation**: They train a lightweight student model (5.5M parameters, 1.4G FLOPs) using pseudo-masks generated by a frozen SAM teacher model. This distillation process eliminates the need for pixel-level ground truth during training while allowing the student model to achieve localization accuracy comparable to fully-supervised methods. The student model directly predicts manipulation regions from input images during inference without requiring prompts.

## Experimental Results
BoxPromptIML achieves a 0.619 F1 score at 20 epochs on in-distribution (IND) datasets (NIST16, CASIAv1, Columbia, Coverage), outperforming other weakly-supervised methods like WSCL (0.239), EdgeCAM (0.322), and SCAF (0.400). On out-of-distribution (OOD) datasets (CocoGlide, In-the-Wild, Korus, DSO, IMD2020), it achieves a 0.403 average F1 score with consistent improvement from 10 to 20 epochs (0.253 to 0.285), demonstrating better generalisation than fully-supervised methods like TruFor (0.267 at 70 epochs) and PSCC-Net (0.323 at 70 epochs). The model's efficiency is notable: it uses 1.4G FLOPs compared to Mesorch's 124.9G FLOPs, representing a 99% reduction in computational cost.

## Related Work
BoxPromptIML positions itself against two main categories of prior work: fully-supervised methods (Mesorch, TruFor, PIM) that require pixel-level annotations and weakly-supervised methods (WSCL, SCAF) that use image-level labels or scribble annotations. Unlike WSCL, which relies on image-level labels (reducing annotation cost to 4 seconds but sacrificing spatial localization), BoxPromptIML uses bounding boxes (7 seconds) to preserve spatial information. It surpasses SCAF, which requires more intensive annotations (around 20 seconds per image) and has higher model complexity, while achieving better efficiency with faster convergence (20 epochs vs. 70 epochs).

## Limitations
The paper doesn't directly compare against other weakly-supervised methods on the same training schedule, making full comparisons difficult. The model's performance on extremely complex manipulations (e.g., AI-generated content with subtle artifacts) isn't thoroughly tested. The authors acknowledge that their method relies on SAM's zero-shot capabilities, which may not generalise to all manipulation types. Additionally, the work focuses primarily on image manipulation localization without addressing other forensics tasks like image generation detection.

## Appendix: Worked Example
Let's walk through how BoxPromptIML processes a single image with concrete numbers:

1. Annotator spends 7 seconds creating a bounding box around the manipulated region in a 512×512 image.
2. The frozen SAM model (with its prompt encoder and mask decoder) processes the image and bounding box, generating a high-quality pseudo-mask (1024×1024 resolution).
3. The student model's Tiny-ViT backbone extracts four multi-scale features (F1-F4) at resolutions 64×64, 32×32, 16×16, and 8×8.
4. The MGFM's Gated Integration computes gate maps using 1×1 convolutions: for example, G1=0.75, G2=0.6, G3=0.45, G4=0.3.
5. These gates guide the fusion: for F1, the preliminary fused feature F''1 = (0.75 ⊙ F1) + (0.25 ⊙ (G2 ⊙ F2 + G3 ⊙ F3 + G4 ⊙ F4)).
6. The Memory Bank contains recalled patterns (e.g., a common splicing artifact pattern with similarity score 0.85).
7. The real-time gate map Gavg = (0.75 + 0.6 + 0.45 + 0.3)/4 = 0.55.
8. The final attention map Afinal = 0.5*(0.65) + 0.5*(0.45) = 0.55 (using α=0.5).
9. This attention map refines the student model's prediction to achieve the final manipulation localization with 0.619 F1 score on IND datasets.
