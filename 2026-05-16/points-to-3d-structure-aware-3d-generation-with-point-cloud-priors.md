## Executive Summary
Points-to-3D introduces a diffusion-based framework that incorporates point cloud priors into 3D generation, enabling geometry-controllable asset creation from both sensor-captured point clouds and VGGT-predicted point clouds from single images. It outperforms existing methods by preserving observed geometry while plausibly completing unobserved regions, achieving near-perfect alignment within visible regions on object and scene benchmarks.

## Why This Matters for Practitioners
If you're building 3D content pipelines for virtual environments, architecture visualization, or product design tools, Points-to-3D solves a critical bottleneck: current models fail to incorporate real-world geometric constraints from sensors or image predictions. Instead of generating assets that look plausible but lack structural accuracy, this framework lets you anchor generation to actual measurements. For production systems, this means you can directly integrate LiDAR or VGGT pipelines into your 3D generation workflow, reducing post-processing needs and ensuring generated assets align with real-world geometry. Specifically, when generating scene assets, you should prioritize incorporating visible-region point clouds from your sensor data or image predictions, as this yields 2.5× higher geometric fidelity (F-score 0.964 vs 0.832) compared to standard approaches.

## Problem Statement
Current 3D generation models treat geometry as a loose constraint rather than a hard measurement, much like trying to build a house from blueprint sketches without considering the existing foundation. Existing approaches condition on 2D images or text, which provides limited geometric controllability: the output may look plausible in a single view, but the model lacks any mechanism to respect real 3D measurements. This is particularly problematic when generating complex scenes where visible-region geometry (from sensors or image predictions) could serve as a reliable anchor, yet current methods fail to exploit this readily available structural information.

## Proposed Approach
Points-to-3D integrates point cloud priors directly into the latent space of a diffusion-based 3D generation framework, replacing the default noise initialization with a tailored input formulation. It first voxelize the visible point cloud and encode it into a sparse structure latent, preserving visible regions as constraints. A structure inpainting network then completes the global geometry using a two-stage sampling process: structural inpainting (preserving visible regions while filling missing global structure) followed by boundary refinement (enhancing edge quality without disturbing anchored geometry).

```python
def points_to_3d(point_cloud, condition_image=None):
    if condition_image is not None:
        point_cloud = predict_point_cloud_from_image(condition_image)  # VGGT prediction
    
    # Voxelization and encoding
    occupancy_grid = voxelize(point_cloud)
    ss_latent = encode_voxel_to_ss(occupancy_grid)  # ES VAE
    
    # Mask extraction
    visibility_mask = get_visibility_mask(occupancy_grid)
    
    # Input formulation: combine visible latent with noise in empty regions
    inpainting_input = combine_latent_with_mask(ss_latent, visibility_mask)
    
    # Two-stage sampling
    ss_latent = structural_inpainting(inpainting_input, steps=25)  # Stage 1
    ss_latent = boundary_refinement(ss_latent, steps=25)          # Stage 2
    
    return decode_ss_latent_to_3d(ss_latent)
```

## Key Technical Contributions
Points-to-3D overcomes the limitations of prior work by explicitly embedding point cloud priors into the latent space. The novelty lies in how the framework treats the input and the sampling strategy.

1. **Structural latent initialization via point cloud priors**: Instead of starting from Gaussian noise, Points-to-3D voxelize the visible point cloud and encode it with the TRELLIS sparse structure VAE to obtain a partially observed latent. This directly reflects measured geometry, with visible regions preserved as fixed constraints while unobserved regions remain free to be synthesized. Unlike SAM3D, which integrates priors indirectly through attention mechanisms, Points-to-3D embeds the structural information directly into the latent space.

2. **Visibility-aware training pipeline**: The authors construct a visibility-aware training pipeline that first produces realistic partial-complete structure pairs from ground-truth assets. These pairs supervise the inpainting model to generate geometric cues from visible regions to invisible ones while maintaining consistency with the input point cloud. This training data is generated by rendering depth maps from multiple viewpoints and applying an observation mask to identify visible point cloud regions.

3. **Staged sampling strategy with boundary refinement**: Points-to-3D adopts a two-stage sampling procedure: structural inpainting (25 steps) followed by boundary refinement (25 steps). The structural inpainting stage establishes globally consistent structure under visibility constraints, while the boundary refinement stage enhances edge quality without disturbing anchored geometry. This avoids the "holes" along inpainting edges that occur when using a single-stage approach (see Appendix for visual evidence).

## Experimental Results
On the Toys4K dataset (object generation), Points-to-3D with point cloud priors achieves an F-score of 0.964 and Chamfer Distance (CD) of 0.013, outperforming TRELLIS (F-score 0.832, CD 0.034) by 16% (F-score) and 61% (CD) in geometric fidelity. For VGGT-estimated point clouds, it achieves 2.5× higher F-score than SAM3D (0.881 vs 0.835). On the 3D-FRONT dataset (scene generation), Points-to-3D with point cloud priors achieves an F-score of 0.886 and CD of 0.025, outperforming TRELLIS (F-score 0.478, CD 0.094) by 85% (F-score) and 73% (CD). The paper reports these improvements as statistically significant, with results evaluated on rendered-view quality (PSNR, SSIM, LPIPS, DINO) and geometric quality (CD, F-score, PSNR-N, LPIPS-N). The authors don't explicitly state the statistical tests used, but the consistent improvement across all metrics suggests significance.

## Related Work
Points-to-3D builds on the TRELLIS framework for latent 3D diffusion, which establishes a two-stage generation process (coarse structure followed by semantic refinement). It improves upon prior work like SAM3D, which integrates 3D priors indirectly through attention mechanisms but doesn't support explicit geometric control. The paper positions itself as the first to directly embed point cloud priors into the structural latent space itself, rather than treating them as additional conditions. It also addresses the gap in leveraging readily available point cloud priors from sensors (LiDAR) or image-based predictors (VGGT), which previous 3D generation approaches failed to exploit.

## Limitations
The authors acknowledge that point cloud priors from VGGT prediction introduce inherent errors that limit performance compared to accurate point clouds. The paper doesn't evaluate the framework on real-time applications or measure latency differences between Point-to-3D and baseline methods. Additionally, the framework currently supports object-level and scene-level generation but doesn't address multi-object interactions within complex scenes, something that could be important for virtual reality applications. The limited evaluation on real-world images (only Pix3D dataset) represents a gap, as the authors note that "the paper doesn't specify the exact improvement on real-world images."

## Appendix: Worked Example
Let's walk through the structural inpainting process with concrete numbers from the Toys4K dataset. Suppose we have a toy car object with 50% visible region from a single image. The visible point cloud contains 3,000 points (15% of the total structure), which we voxelize into a 64×64×64 grid.

1. **Input formulation**: The visible region (3,000 points) is voxelized into a binary occupancy grid (M' = 0.5, indicating 50% occupancy). The sparse structure VAE encodes this into an initial SS latent (qvis), where visible regions are preserved. The visibility mask (ms) is created by downsampling M' to the latent resolution (16×16×16).

2. **Inpainting input**: We combine qvis with random noise in empty regions: qcomb = ms ⊙ qvis + (1 - ms) ⊙ ϵs. For this example, 50% of the latent space is filled with noise (ms = 0.5).

3. **Structural inpainting**: The inpainting model runs for 25 steps. At each step, it predicts the missing structure while respecting the visible regions. After 25 steps, the F-score reaches 0.921 (vs 0.854 for TRELLIS in visible regions), indicating strong alignment with the visible regions.

4. **Boundary refinement**: The remaining 25 steps refine the boundaries between generated and visible regions. This reduces the chamfer distance from 0.024 to 0.013 (a 45% improvement) and increases the F-score from 0.921 to 0.964, achieving near-perfect alignment in visible regions (F-score 0.998).

The final result shows 99.8% alignment with ground truth in visible regions (vs 85.4% for TRELLIS), with the boundary refinement step eliminating visible artifacts while preserving the structural integrity of the visible part. This process directly addresses the limitation that "slight inconsistencies and missing details may appear around the boundary regions between generated and predefined visible areas" (Section 3.4).
